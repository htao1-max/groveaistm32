# 2026-03-13 Update: i2ccomm SD-log Recording Trigger

## What Changed

Replaced the SSCMA AT-command driver with a raw Himax `i2ccomm` packet sender in `Core/Src/main.c`.

The Grove Vision AI Module V2 now runs custom firmware (`tflm_yolov8_od_sdlog`) that:
- Runs YOLOv8n object detection continuously
- Waits for an I2C command to start saving frames to SD card
- Saves ALL frames to `SESSION_XXXX/ALL/` and detection-only frames to `SESSION_XXXX/DETECT/`

The STM32 acts as the I2C master that triggers recording by sending a single i2ccomm packet.

### Changes in `main.c`

- Removed `Seeed_Arduino_SSCMA.h` include and all SSCMA API calls
- Added `crc16_ccitt()` — CCITT CRC-16 checksum (poly 0x1021, init 0xFFFF), matching the Himax i2c_comm library
- Added `grove_start_recording(int threshold)` — builds and sends an i2ccomm packet:
  - Feature: `0x80` (customer range)
  - Command: `0x01` (start recording)
  - Payload: 1 byte threshold (0-100, maps to 0.00-1.00 on Grove side)
  - Checksum: 2-byte CRC-16 appended at the end
- On boot: waits 5s for Grove AI to finish init, then sends start-recording with 50% threshold
- Main loop: LED heartbeat on PB9 every 2s

### i2ccomm Packet Format

```
Byte 0: Feature (0x80)
Byte 1: Command (0x01)
Byte 2: Payload length LSB
Byte 3: Payload length MSB
Byte 4: Payload (threshold 0-100)
Byte 5: CRC16 LSB
Byte 6: CRC16 MSB
```

## How to Test

### Prerequisites

- Grove Vision AI V2 flashed with `tflm_yolov8_od_sdlog` firmware + YOLOv8n model
- FAT32-formatted microSD card inserted into Grove AI V2
- STM32G431 connected to Grove AI V2 via I2C1 (PB7=SDA, PB8=SCL)
- OV5647 camera connected to Grove AI V2

### Steps

1. **Build & flash STM32**: Open this project in STM32CubeIDE, build, and flash to the STM32.

2. **Open two serial monitors**:
   - **STM32 UART**: 115200 baud, 8N1 (e.g. PuTTY on the ST-Link COM port)
   - **Grove AI UART**: 921600 baud, 8N1 (e.g. PuTTY on the Grove USB-C COM port)

3. **Power on / reset both boards**.

4. **Expected STM32 UART output**:
   ```
   [0000000000 ms] ========================================
   [0000000000 ms]   STM32 + Grove AI (i2ccomm SD-log)
   [0000000000 ms] ========================================
   [0000000001 ms] I2C: Grove AI found at 0x62
   [0000000001 ms] Waiting 5s for Grove AI to finish init...
   [0000005001 ms] Sending start-recording (threshold=50%)...
   [0000005002 ms] I2C TX OK — recording should start on Grove AI
   [0000005002 ms] Done. Check Grove AI UART for [I2C_CMD] message.
   [0000005002 ms] SD card will log to SESSION_XXXX/ALL/ and DETECT/
   ```

5. **Expected Grove AI UART output** (among the JSON stream):
   ```
   [I2C_CMD] Recording started (threshold=0.50)
   ```

6. **Let it run for 10-30 seconds**, then power off the Grove AI.

7. **Check SD card on PC**:
   - `SESSION_XXXX/ALL/` — should contain `img_0000.jpg`, `img_0001.jpg`, etc.
   - `SESSION_XXXX/DETECT/` — should contain `det_XXXX.jpg` files for frames with detections above 50% confidence
   - `SESSION_XXXX/session.log` — text log of all saves and detections

### Troubleshooting

| Symptom | Fix |
|---|---|
| STM32 shows "I2C: Grove AI NOT found" | Check I2C wiring (PB7=SDA, PB8=SCL), ensure Grove AI is powered |
| STM32 shows "I2C TX FAILED" | Grove AI may not have finished booting — increase the 5s delay |
| Grove AI shows "checksum error" | CRC mismatch — verify the CRC-16 implementation matches CCITT standard |
| No `[I2C_CMD]` message on Grove AI | The i2ccomm event handler may not be running yet — increase delay |
| SD card empty | Recording didn't trigger — check Grove AI log for the `[I2C_CMD]` line |
| `DETECT/` empty | No detections above threshold — point camera at a person or COCO object, or lower threshold |
