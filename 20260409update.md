# 0409 Update — YOLOv8n Model Export for Grove Vision AI V2

## What was done

### 1. Model Training (DONE)
- Trained YOLOv8n object detection model on Ultralytics Hub
- 2 classes: BlueRing, RedRing
- Input size: 192x192
- Downloaded trained weights as `exp.pt` (renamed from `best.pt` by Hub)

### 2. Export to int8 TFLite (DONE)
- Exported `exp.pt` to `*_full_integer_quant.tflite` using Ultralytics export API
- Used `coco128.yaml` for int8 quantization calibration (actual dataset had missing valid/images folder)
- Also generated `exp-2.onnx` for the delete-transpose step

### 3. Re-export with Ultralytics v8.0.173 (DONE)
- Newer Ultralytics versions produce different ONNX graph structures incompatible with the HimaxWiseEyePlus guide
- Created venv with `ultralytics==8.0.173` + `torch==2.0.1` to match the guide
- Re-exported ONNX — confirmed graph structure now has cv2+cv3 concatenated per detection head (66 = 64 bbox + 2 classes)

### 4. Param Replacement JSON (DONE)
- Mapped all 10 ONNX node entries using Netron:
  - 3 Reshapes: `/model.22/Reshape`, `/model.22/Reshape_1`, `/model.22/Reshape_2`
  - 1 Concat: `/model.22/Concat_3`
  - 1 dfl Reshape: `/model.22/dfl/Reshape`
  - 1 dfl Transpose: `/model.22/dfl/Transpose`
- Saved as `replace_192_2cls_transpose_op.json`
- All `144` values (from 80-class example) changed to `66` (64 + 2 classes)

### 5. onnx2tf Conversion — NOT YET COMPLETE
- Attempted on Windows — massive dependency conflicts between tensorflow, numpy, onnx2tf, tf_keras, etc.
- The guide targets **Ubuntu 20.04 LTS** — Windows is not supported
- Set up Ubuntu VM in VirtualBox with shared folder access to Windows files
- Created Python 3.10 venv in `~/himax_venv` (shared folders don't support symlinks)
- Installed base packages, ready to install the full dependency chain
