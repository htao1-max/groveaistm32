# 2026-04-03 Update: Extract Grove Himax Code into himax_sdk Module

## What Was Done

Refactored `main.c` to move all Grove Himax-related code into dedicated `himax_sdk.h` / `himax_sdk.c` files for cleaner separation.

### Files Created / Modified

**`Core/Inc/himax_sdk.h`** (new proper header):
- Added include guard (`#ifndef INC_HIMAX_SDK_H_`)
- Moved `#define`s here: `GROVE_I2C_ADDR`, `I2C_FEATURE_RECORDER`, `I2C_CMD_RECORD_START`
- Declared public API: `uart_log()`, `initForHimax()`, `startRecordingForHimax()`

**`Core/Src/himax_sdk.c`** (updated):
- Removed `static` from public functions (`uart_log`, `initForHimax`, `startRecordingForHimax`)
- Added `extern` declarations for `hi2c1` and `huart4` (HAL handles defined in `main.c`)
- Added `<stdarg.h>` include for `va_list` used by `uart_log`
- `crc16_ccitt` and `grove_start_recording` remain `static` (internal helpers)

**`Core/Src/main.c`** (cleaned up):
- Removed duplicate `crc16_ccitt()`, `grove_start_recording()`, `uart_log()` functions
- Removed duplicate `#define`s (`GROVE_I2C_ADDR`, `I2C_FEATURE_RECORDER`, `I2C_CMD_RECORD_START`)
- Removed duplicate inline I2C bus scan code that was duplicating `initForHimax()` logic
- `USER CODE BEGIN 2` now just calls `initForHimax()` → `startRecordingForHimax()`
- Only includes `"himax_sdk.h"` (removed now-unnecessary `<stdio.h>`, `<stdarg.h>`, `<string.h>`)

### Issue Encountered

After refactoring, the Grove AI device was not found on the I2C bus (0 devices in scan). The old `main.c` had the I2C scan running twice — once inline at 1.5s delay, then again inside `initForHimax()` at 15s delay. The first scan likely "warmed up" the bus. With only `initForHimax()` running, the single scan after 15s was NACKing. Boot delay was adjusted to 5s.
