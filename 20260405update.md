# 2026-04-05 Update: Camera Resolution & OV5647 Controls Investigation

## What Was Done

### Image Resolution Investigation

Attempted to increase SD card image resolution from 320x240 to 640x480 by changing the datapath init parameter in `tflm_yolov8_od_sdlog.c` (lines 549, 560, 570) from `APP_DP_RES_RGB640x480_INP_SUBSAMPLE_2X` to `APP_DP_RES_RGB640x480_INP_SUBSAMPLE_1X`.

**Result: BusFault crash** on the Grove AI module:
```
Entering BusFault_Handler interrupt!
SCB->CFSR:0x00008200
SCB->BFAR:0x00040000
```
- Cause: `_SUBSAMPLE_1X` requires 921,600 bytes for the raw frame buffer (`cisdp_sensor.c:89`) vs 230,400 bytes for `_SUBSAMPLE_2X` (line 99). The Himax WE2 only has ~512KB SRAM total, and the YOLO model already consumes most of it.
- The faulting address `0x00040000` (256KB boundary) confirms the DMA write overflowed available SRAM.
- **Decision**: Reverted to `_SUBSAMPLE_2X` (320x240). Full 640x480 RGB is not feasible with YOLOv8 running concurrently. YUV format (`APP_DP_RES_YUV640x480_INP_SUBSAMPLE_1X`, 460KB) could be a possible alternative but was not tested.

### SD Card Frame Rate

Confirmed the SD card recording has no software FPS limiter. Every frame from the sensor triggers `EVT_INDEX_XDMA_FRAME_READY` in `tflm_yolov8_od_sdlog.c:411`, and when `g_recording_active == 1`, every frame is saved. Effective frame rate = sensor FPS minus SD write + YOLO inference overhead.

### OV5647 Camera Controls Documentation

Created `OV5647_Camera_Controls.md` in the Grove AI repo root documenting all controllable camera parameters via register writes in `OV5647_mipi_2lane_640x480.i`:
- Exposure: auto/manual, AEC brightness targets, anti-flicker banding (50/60Hz)
- Gain: gain ceiling, manual gain control
- Image orientation: mirror and flip via `0x3820`/`0x3821`
- Frame rate: PLL clock dividers, blanking timing
- Window/crop: output size, ISP input crop, subsampling
- ISP processing: white balance, lens shading, sharpness
- Test pattern: color bar for debugging
