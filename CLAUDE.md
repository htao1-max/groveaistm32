# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

This is an STM32G431CBUx (STM32G4 series) embedded C/C++ project targeting a Grove AI Vision module (Seeed SSCMA). The main application performs face detection via an I2C-connected Grove AI camera module and toggles GPIO PB9 when a face is detected, with debug output over UART4 at 115200 baud.

## Build System

This project is built exclusively through **STM32CubeIDE** (Eclipse-based). There is no command-line Makefile workflow intended for regular use. Build artifacts land in `Debug/`.

- **Toolchain**: GCC (arm-none-eabi-gcc), managed by STM32CubeIDE
- **Linker script**: `STM32G431CBUX_FLASH.ld`
- **CubeMX config**: `i2cScan.ioc` — regenerate HAL init code by opening in STM32CubeIDE and clicking "Generate Code". Do not manually edit generated sections.
- **Launch config**: `i2cScan Debug.launch` — used by STM32CubeIDE for debug sessions via ST-Link

To build from the command line (if needed):
```sh
# From the Debug/ directory (generated by CubeMX/CubeIDE):
make -C Debug all
```

## Hardware Configuration

- **MCU**: STM32G431CBUx @ 16 MHz (HSI, no PLL active)
- **I2C1** (PB7=SDA, PB8=SCL, timing `0x00503D58`): connected to Grove AI Vision module at I2C address `0x62` (8-bit: `0xC4`)
- **I2C2** (PA8=SDA, PA9=SCL): available but unused in current application
- **UART4** (PC10=TX, PC11=RX): debug output at 115200 baud, TX-only mode
- **PB9**: GPIO output — toggled on face detection

## Architecture

### SSCMA Driver (`Core/Seeed_Arduino_SSCMA/src/`)

This is a custom C++ STM32 HAL port of the Seeed Arduino SSCMA library. It is the active driver:

- **`Seeed_Arduino_SSCMA.h`** — public API. Key types: `SSCMA_t` (main device handle), `boxes_t`, `classes_t`, `perf_t`.
- **`Seeed_Arduino_SSCMA.cpp`** — implementation. Despite the `.cpp` extension, the public API is wrapped in `extern "C"` blocks so it links with the C `main.c`.

Key API functions:
- `SSCMA_Init_I2C()` — initializes the device, allocates RX/TX buffers (4KB each via `malloc`), verifies connectivity by querying ID and Name.
- `SSCMA_SelectModel(sscma, id)` — sends `AT+MODEL=<id>` to load a model on the Grove AI module (model 1 = face detection).
- `SSCMA_InvokeOnce(sscma)` — sends `AT+INVOKE=1,0,1`, waits for an EVENT response, parses the `"boxes"` field. Sets `sscma.box_detected = 1` if a bounding box is present.
- `wait_response()` — internal polling loop that reads I2C data until a `\r{...}\n` JSON response matching the expected type and command name arrives, or times out.
- `parse_event()` — minimal JSON parser (manual `strstr`/`sscanf`) for INVOKE events; populates `sscma.last_box`.

### Main Application (`Core/Src/main.c`)

The application flow:
1. HAL init → clock config → peripheral init (GPIO, UART4, I2C1, I2C2)
2. Probe I2C address `0x62`, report over UART4
3. `SSCMA_Init_I2C()` → get name/ID → `SSCMA_SelectModel(1)` (300ms delay)
4. Main loop: `SSCMA_InvokeOnce()` every ~150ms → toggle PB9 and print `"Face detected\r\n"` if `sscma.box_detected`

### AT Command Protocol

The Grove AI module uses a text AT-command protocol over I2C:
- Commands: `AT+<CMD>\r\n`
- Responses: `\r{<JSON>}\n`
- JSON fields: `"type"` (0=response, 1=event, 2=log), `"name"`, `"data"`, `"boxes"` (array of `[x,y,w,h,score,target]`)
- I2C transport uses a 6-byte framing protocol with `FEATURE_TRANSPORT` (0x10) commands for read/write/available queries

## STM32CubeIDE Code Generation Rules

STM32CubeMX generates `main.c` and HAL init files. User code must only go inside designated comment blocks:
```c
/* USER CODE BEGIN <tag> */
// your code here
/* USER CODE END <tag> */
```
Code outside these blocks will be overwritten on next code generation. The SSCMA driver in `Core/Seeed_Arduino_SSCMA/` is a manually added subfolder and is not managed by CubeMX.
